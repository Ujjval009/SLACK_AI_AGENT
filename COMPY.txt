1. PROJECT OVERVIEW
================================================================================

  This is a Slack AI Agent that automatically analyzes new members who join
  a Slack workspace. When someone joins (via team_join or member_joined_channel
  events), the agent:

   1. Fetches their Slack profile (name, email, title, timezone, department,
      team, phone, display name, status text, skype)
   2. Performs basic research:
      - Extracts rich profile fields from Slack
      - Optionally searches DuckDuckGo for the person's name and title
   3. Sends the profile + research data to Groq's Llama 3.3 70B model
  4. The LLM returns a JSON analysis with:
     - fitScore (0-100): How likely they'd be interested in CloudNest Deploy
     - insights: 3-5 key observations
     - recommendations: 2-4 engagement suggestions
  5. Saves the analysis to PostgreSQL
  6. Posts a rich Slack message (with color-coded score) to a private channel

  TECH STACK:
  - FastAPI (async Python web framework)
  - Uvicorn (ASGI server)
  - LangChain (LLM orchestration)
  - langchain-groq (Groq provider for LangChain)
  - Slack SDK (Socket Mode client + Web API client)
  - SQLAlchemy 2.0 (async ORM)
  - asyncpg (async PostgreSQL driver)
  - httpx (async HTTP client for research)
  - Pydantic (validation & settings)
  - python-dotenv (env file loading)

3. COMPLETE DATA PIPELINE (Step by Step)
================================================================================

  EVENT TRIGGER
  ─────────────
  1. Slack workspace fires a 'team_join' or 'member_joined_channel' event
  2. Socket Mode client (persistent WebSocket) receives it in real-time
  3. handle_socket_mode_request() inspects the event type

  PROFILE FETCH
  ─────────────
  4. get_user_info(user_id) calls Slack API:
     - GET https://slack.com/api/users.info?user={id}
     - Returns: name, email, title, timezone, profile fields
  5. Maps Slack's raw response into a clean Pydantic MemberInfo

   RESEARCH PHASE
   ─────────────
   6. do_basic_research(member_info) extracts:
      - Slack profile fields: department, team, phone, displayName,
        skype, statusText, timezone
      - Builds profile insights from available fields

   7. Web search (optional):
      - HTTP GET https://lite.duckduckgo.com/lite/?q={name}+{title}
      - Returns search results if page is non-empty
      - Gracefully fails (None) on timeout or error

   8. Research results collected as list[ResearchResult]
      - type: "profile" for Slack fields, "web_search" for web results

  AI ANALYSIS
  ───────────
  10. analyze_with_ai(member_info, research_data):
      - Builds a ChatPromptTemplate with:
        • Company/Product context
      • Member details (name, email, title, timezone, profile fields)
      • Research findings (Slack profile insights + web search)
      - Chains: prompt | ChatGroq(model="llama-3.3-70b-versatile")
      - Calls llm.ainvoke() with formatted variables
      - Groq returns Llama 3.3 70B's response text
      - Strips ```json markers, parses JSON

  11. Parsed fields extracted:
      - fitScore (clamped to 0-100)
      - insights (list of strings)
      - recommendations (list of strings)

  DATABASE PERSISTENCE
  ────────────────────
  12. save_member_analysis():
      - INSERT INTO member_analyses (member_id, member_name, member_email,
        member_title, member_timezone, fit_score, insights, recommendations,
        research_data) VALUES (...)
      - Returns auto-generated ID

  SLACK POSTING
  ─────────────
  13. _build_slack_blocks(member, analysis):
      - Color-coded attachment:
        • score >= 80 → green (#36a64f)
        • score >= 60 → amber (#ffb84d)
        • score >= 40 → orange (#ff9500)
        • score < 40  → red   (#ff4444)
      - Header: "🔍 New Member: {name}"
      - Fields: Fit Score, Email, Title
      - Section: Insights (bullet points)
      - Section: Recommendations (bullet points)
      - Footer: Timestamp

  14. post_analysis_to_channel():
      - POST https://slack.com/api/chat.postMessage
      - Channel: SLACK_PRIVATE_CHANNEL_ID
      - Payload: text + rich attachment blocks

  15. mark_as_sent_to_slack():
      - UPDATE member_analyses SET sent_to_slack = TRUE,
        sent_to_slack_at = NOW(), updated_at = NOW() WHERE id = {id}

  ERROR HANDLING
  ──────────────
   - If any step fails (research, LLM, Slack post), the error is logged
   - The analysis record remains in DB (if saved before failure)
   - LLM defaults to fitScore: 50 when data is limited — "unknown" ≠ "bad"
  - Graceful shutdown on SIGINT/SIGTERM

8. TESTING GUIDE
================================================================================

  TEST 1: Health Check
  ────────────────────
  curl http://localhost:8000/health

  Expected:
  {"status":"healthy","timestamp":"2026-06-06T07:00:00.000000+00:00"}

  TEST 2: AI Analysis Pipeline (No Slack/DB required)
  ────────────────────────────────────────────────────
  curl -X POST http://localhost:8000/test/analyze-member \
    -H "Content-Type: application/json" \
    -d '{
      "memberInfo": {
        "name": "Jane Doe",
        "email": "jane@google.com",
        "title": "VP Engineering"
      }
    }'

  Expected:
  {"success":true,"analysis":{"fitScore":90,"insights":[...],"recommendations":[...]},"timestamp":"..."}

  TEST 3: Full Live Flow (Requires Slack tokens)
  ──────────────────────────────────────────────
  1. Invite your bot to a public channel
  2. Have a new user join the Slack workspace
  3. Bot automatically posts analysis to SLACK_PRIVATE_CHANNEL_ID

   TEST 4: Edge Cases
   ──────────────────
   - Personal email (jane@gmail.com) → AI does NOT penalize for this
   - No email provided → AI uses "Not provided"
   - No job title → AI infers from context or defaults to score 50
   - LLM returns bad JSON → Falls back to fitScore=50, manual review
   - Research endpoint down → Continues with available data
   - Minimal web presence → AI defaults to score 50 ("unknown" ≠ "bad")

9. RENDER DEPLOYMENT
================================================================================

   MANUAL DEPLOYMENT STEPS:
   ────────────────────────
   1. Push code to GitHub
   2. In Render Dashboard → "New +" → "Web Service"
   3. Connect your repository
   4. Set:
      - Name: slack-ai-agent
      - Runtime: Python 3
      - Build Command: pip install -r requirements.txt
      - Start Command: uvicorn app.main:app --host 0.0.0.0 --port $PORT
   5. Add all environment variables in Render's dashboard
   6. Deploy (free tier works without credit card)

   IMPORTANT NOTES:
   - render.yaml was removed — Blueprint deployment requires a credit card
     even for free tier. Use manual Web Service instead.
   - Set DATABASE_URL with `connect_args={"ssl": "require"}` in database.py
     (asyncpg does NOT support `?sslmode=require` query parameter)
   - Pin Python version in runtime.txt (3.12.0) — Render defaulted to 3.14.3
     which caused Rust compilation issues for pydantic-core
   - Unpin package versions in requirements.txt to get pre-built wheels
     (avoids Rust compilation on Render's read-only filesystem)
   - The start command uses $PORT which Render sets automatically
